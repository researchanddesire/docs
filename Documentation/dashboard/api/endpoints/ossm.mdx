## List OSSM Devices

<div className="not-prose">`GET /api/v1/ossm`</div>

Returns all OSSM devices you have access to.

<ParamField query="limit" type="number" default="50">
  Number of records to return (max 100).
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip for pagination.
</ParamField>

<ResponseField name="data.data" type="object[]" required>
  Array of OSSM device objects.

<ResponseField name="id" type="number">
  Device ID.
</ResponseField>

<ResponseField name="bubbleId" type="string">
  Unique device identifier.
</ResponseField>

<ResponseField name="macAddress" type="string">
  Device MAC address.
</ResponseField>

<ResponseField name="chip" type="string">
  Hardware chip identifier.
</ResponseField>

<ResponseField name="ownerId" type="number">
  ID of the device owner.
</ResponseField>

<ResponseField name="lastVisited" type="string">
  ISO 8601 timestamp of the last device connection.
</ResponseField>

  <ResponseField name="OSSMFirmwareId" type="number">
    ID of the currently installed firmware version.
  </ResponseField>
</ResponseField>

<ResponseField name="data.pagination" type="object" required>
  Pagination metadata.

<ResponseField name="limit" type="number">
  Number of records returned.
</ResponseField>

  <ResponseField name="offset" type="number">
    Number of records skipped.
  </ResponseField>
</ResponseField>

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dashboard.researchanddesire.com/api/v1/ossm?limit=50&offset=0"
```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "data": [
      {
        "id": 1,
        "bubbleId": "ossm-xyz",
        "macAddress": "11:22:33:44:55:66",
        "chip": "ESP32-S3",
        "ownerId": 42,
        "lastVisited": "2026-02-17T09:15:00Z",
        "OSSMFirmwareId": 5
      }
    ],
    "pagination": {
      "limit": 50,
      "offset": 0
    }
  }
}
```
</ResponseExample>

## Get OSSM Device

<div className="not-prose">`GET /api/v1/ossm/{id}`</div>

Returns full details for a specific OSSM device.

<ParamField path="id" type="number" required>
  The OSSM device ID.
</ParamField>

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://dashboard.researchanddesire.com/api/v1/ossm/1
```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "id": 1,
    "bubbleId": "ossm-xyz",
    "macAddress": "11:22:33:44:55:66",
    "chip": "ESP32-S3",
    "ownerId": 42,
    "lastVisited": "2026-02-17T09:15:00Z",
    "OSSMFirmwareId": 5,
    "pairingCode": null,
    "pairingExpires": null
  }
}
```

```json 404 Not Found
{
  "ok": false,
  "error": "OSSM device not found"
}
```

</ResponseExample>
