## List Users

<div className="not-prose">`GET /api/v1/users`</div>

Returns all users you have access to. Results are filtered by your account permissions — you'll see yourself and any users connected to you via partnerships. Includes profile information in a flat structure.

<ParamField query="limit" type="number" default="50">
  Number of records to return (max 100).
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip for pagination.
</ParamField>

<ResponseField name="ok" type="boolean" required>
  Always `true` on success.
</ResponseField>

<ResponseField name="data.data" type="object[]" required>
  Array of user objects with profile information.

<ResponseField name="id" type="number">
  The user's internal ID.
</ResponseField>

<ResponseField name="username" type="string">
  The user's unique username (from profile).
</ResponseField>

<ResponseField name="accountType" type="string">
  One of `user`, `support`, `admin`, or `superadmin`.
</ResponseField>

<ResponseField name="subscriptionTier" type="string">
  Subscription tier: `ultra`, `pioneer`, `founder`, `employee`, or `null` if no subscription.
</ResponseField>

<ResponseField name="league" type="number">
  The user's current league tier.
</ResponseField>

<ResponseField name="rank" type="number">
  The user's leaderboard rank.
</ResponseField>

<ResponseField name="totalPoints" type="number">
  Lifetime points earned.
</ResponseField>

<ResponseField name="matchPoints" type="number">
  Points earned in the current match period.
</ResponseField>

<ResponseField name="profileImage" type="string">
  URL to the user's profile image (nullable).
</ResponseField>

<ResponseField name="profileDescription" type="string">
  User's profile bio/description (nullable).
</ResponseField>

  <ResponseField name="profileTitle" type="string">
    User's profile title (nullable).
  </ResponseField>
</ResponseField>

<ResponseField name="data.pagination" type="object" required>
  Pagination metadata.

  <ResponseField name="limit" type="number">
    Number of records returned.
  </ResponseField>

  <ResponseField name="offset" type="number">
    Number of records skipped.
  </ResponseField>
</ResponseField>

<RequestExample>
  ```bash cURL curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dashboard.researchanddesire.com/api/v1/users?limit=50&offset=0" ```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "data": [
      {
        "id": 42,
        "username": "alice_trainer",
        "accountType": "user",
        "subscriptionTier": "ultra",
        "league": 3,
        "rank": 150,
        "totalPoints": 12500,
        "matchPoints": 800,
        "profileImage": "https://example.com/avatar.jpg",
        "profileDescription": "Hello world",
        "profileTitle": "Expert Trainer"
      }
    ],
    "pagination": {
      "limit": 50,
      "offset": 0
    }
  }
}
```
</ResponseExample>

## Get User by ID

<div className="not-prose">`GET /api/v1/users/{id}`</div>

Returns detailed information about a specific user, including their profile data. You can only access users you have a connection with.

<ParamField path="id" type="number" required>
  The user's internal ID.
</ParamField>

<ResponseField name="data" type="object" required>
  User object with extended fields.

<ResponseField name="id" type="number">
  The user's internal ID.
</ResponseField>

<ResponseField name="username" type="string">
  The user's unique username.
</ResponseField>

<ResponseField name="accountType" type="string">
  Account type.
</ResponseField>

<ResponseField name="subscriptionTier" type="string">
  Subscription tier.
</ResponseField>

<ResponseField name="totalPoints" type="number">
  Lifetime points earned.
</ResponseField>

<ResponseField name="matchPoints" type="number">
  Points earned in the current match period.
</ResponseField>

  <ResponseField name="Profile" type="object">
    The user's public profile, if accessible.
  </ResponseField>
</ResponseField>

<RequestExample>
  ```bash cURL curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://dashboard.researchanddesire.com/api/v1/users/42 ```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "id": 42,
    "username": "alice_trainer",
    "accountType": "user",
    "subscriptionTier": "ultra",
    "league": 3,
    "rank": 150,
    "totalPoints": 12500,
    "matchPoints": 800,
    "profileImage": "https://example.com/avatar.jpg",
    "profileDescription": "Hello world",
    "profileTitle": "Expert Trainer"
  }
}
```

```json 404 Not Found
{
  "ok": false,
  "error": "User not found"
}
```

</ResponseExample>
