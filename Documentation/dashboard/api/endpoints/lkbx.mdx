## List Lockbox Devices

<div className="not-prose">`GET /api/v1/lkbx`</div>

Returns all Chastity Lockbox devices you have access to.

<ParamField query="limit" type="number" default="50">
  Number of records to return (max 100).
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip for pagination.
</ParamField>

<ResponseField name="data.data" type="object[]" required>
  Array of lockbox device objects.

<ResponseField name="id" type="number">
  Device ID.
</ResponseField>

<ResponseField name="bubbleId" type="string">
  Unique device identifier.
</ResponseField>

<ResponseField name="macAddress" type="string">
  Device MAC address.
</ResponseField>

<ResponseField name="ownerId" type="number">
  ID of the device owner.
</ResponseField>

<ResponseField name="locked" type="boolean">
  Whether the device is currently in a locked state.
</ResponseField>

<ResponseField name="hueShift" type="number">
  LED color customization value.
</ResponseField>

  <ResponseField name="lastVisited" type="string">
    ISO 8601 timestamp of the last device connection.
  </ResponseField>
</ResponseField>

<ResponseField name="data.pagination" type="object" required>
  Pagination metadata.

<ResponseField name="limit" type="number">
  Number of records returned.
</ResponseField>

  <ResponseField name="offset" type="number">
    Number of records skipped.
  </ResponseField>
</ResponseField>

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dashboard.researchanddesire.com/api/v1/lkbx?limit=50&offset=0"
```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "data": [
      {
        "id": 1,
        "bubbleId": "lkbx-abc",
        "macAddress": "AA:BB:CC:DD:EE:FF",
        "ownerId": 42,
        "locked": true,
        "hueShift": 120,
        "lastVisited": "2026-02-17T10:30:00Z"
      }
    ],
    "pagination": {
      "limit": 50,
      "offset": 0
    }
  }
}
```
</ResponseExample>

## Get Lockbox Device

<div className="not-prose">`GET /api/v1/lkbx/{id}`</div>

Returns full details for a specific lockbox device.

<ParamField path="id" type="number" required>
  The lockbox device ID.
</ParamField>

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://dashboard.researchanddesire.com/api/v1/lkbx/1
```
</RequestExample>

## List Lock Templates

<div className="not-prose">`GET /api/v1/lkbx/templates`</div>

Returns all lock templates you have access to. These templates define how a lock session behaves.

<ParamField query="limit" type="number" default="50">
  Number of records to return (max 100).
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip for pagination.
</ParamField>

<ResponseField name="data.data" type="object[]" required>
  Array of lock template objects.

<ResponseField name="id" type="number">
  Template ID.
</ResponseField>

<ResponseField name="name" type="string">
  Template name.
</ResponseField>

<ResponseField name="duration" type="number">
  Lock duration in seconds.
</ResponseField>

<ResponseField name="isRandomDuration" type="boolean">
  Whether duration is randomized within a range.
</ResponseField>

<ResponseField name="isBreakEnabled" type="boolean">
  Whether periodic breaks are allowed.
</ResponseField>

<ResponseField name="isEmergencyUnlockEnabled" type="boolean">
  Whether emergency unlock is available.
</ResponseField>

<ResponseField name="isTestLock" type="boolean">
  Whether this is a test lock (anyone can unlock).
</ResponseField>

  <ResponseField name="isTimeDisplayed" type="boolean">
    Whether remaining time is shown to the wearer.
  </ResponseField>
</ResponseField>

<ResponseField name="data.pagination" type="object" required>
  Pagination metadata.

<ResponseField name="limit" type="number">
  Number of records returned.
</ResponseField>

  <ResponseField name="offset" type="number">
    Number of records skipped.
  </ResponseField>
</ResponseField>

<RequestExample>
```bash cURL
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dashboard.researchanddesire.com/api/v1/lkbx/templates?limit=50&offset=0"
```
</RequestExample>

<ResponseExample>
```json 200 OK
{
  "ok": true,
  "data": {
    "data": [
      {
        "id": 3,
        "name": "Weekend Lock",
        "duration": 172800,
        "isRandomDuration": false,
        "isBreakEnabled": true,
        "isEmergencyUnlockEnabled": true,
        "isTestLock": false,
        "isTimeDisplayed": true
      }
    ],
    "pagination": {
      "limit": 50,
      "offset": 0
    }
  }
}
```
</ResponseExample>
